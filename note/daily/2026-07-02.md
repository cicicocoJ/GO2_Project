# 2026-07-02 GO2 二次开发工作总结

## 1. 今日总体进展

今天主要围绕 GO2 巡检系统的感知模块、网页控制台优化和工程化启动流程展开。在已有运动控制、姿态控制、后台桥接和状态上传链路的基础上，新增并验证了 D435i 相机拍照、实时视频显示，以及 XT-16 激光雷达点云接入能力。

当前系统已从“网页控制机器人运动”进一步扩展为“网页控制 + 实时视频 + 拍照采集 + 激光点云输入”的巡检控制台雏形。

---

## 2. D435i 相机拍照节点完成

新建 ROS2 包：

```text
go2_camera_capture
```

完成节点：

```text
camera_capture_node
```

主要功能：

```text
订阅 /camera_capture_command 或 /backend_command
收到 CAPTURE_IMAGE 命令
读取 D435i 彩色图像流 /dev/video4
保存 JPG 图片到 /home/unitree/go2_captures
发布 /camera_capture_result 拍照结果
```

经过测试，D435i 在 Jetson 上识别正常：

```text
/dev/video4 = 彩色 RGB 图像
/dev/video2 = 黑白/红外图像
```

拍照命令已经测试成功，图片能够正常保存。

---

## 3. Dashboard 增加拍照按钮

在 FastAPI Dashboard 中加入了拍照按钮：

```text
CAPTURE_IMAGE
```

当前链路为：

```text
网页 Dashboard 点击拍照
    ↓
FastAPI 后台 command 接口
    ↓
backend_client_node
    ↓
/backend_command
    ↓
camera_capture_node
    ↓
D435i 拍照并保存图片
```

至此，网页端已经可以触发 Jetson 上的 D435i 拍照功能。

## 4. D435i 实时视频流接入网页

新增 Jetson 侧 MJPEG 视频流服务：

```text
go2_camera_stream_server.py
```

主要功能：

```text
OpenCV 读取 /dev/video4
编码为 JPEG
通过 HTTP MJPEG 输出实时视频流
Dashboard 使用 img 标签显示实时画面
```

视频流地址：

```text
http://192.168.123.18:8081/
http://192.168.123.18:8081/video_feed
```

已经解决网页中视频显示被裁剪的问题，当前实时画面显示正常。

## 6. 一键启动脚本整理

为减少每次调试需要打开多个终端的问题，设计了“一键启动 / 停止”脚本方案。

Jetson 侧脚本：

```text
scripts/start_robot_side.sh
scripts/stop_robot_side.sh
```

笔记本侧脚本：

```text
scripts/start_all_go2_system.sh
scripts/stop_all_go2_system.sh
```

计划一键启动内容包括：

```text
FastAPI 后台
backend_client_node
go2_state_reader_node
backend_command_handler_node
camera_capture_node
D435i MJPEG 视频流服务
```

该部分主要用于后续工程化运行，减少手动启动复杂度。

## 7. XT-16 激光雷达接入与点云读取

今日将禾赛 XT-16 激光雷达接入 Jetson，并确认其不是 USB 设备，而是通过 M8 航插线束转以太网 UDP 通信。

通过 Python 抓包确认 Jetson 已经收到 XT-16 UDP 数据：

```text
192.168.123.20:10000 -> 192.168.123.18:2368
```

说明：

```text
XT-16 雷达 IP：192.168.123.20
Jetson IP：192.168.123.18
点云 UDP 端口：2368
```

随后下载并编译 Hesai ROS2 驱动：

```text
HesaiLidar_ROS_2.0
HesaiLidar_SDK_2.0
```

编译过程中解决了 SDK 子模块缺失问题，最终编译成功。

启动驱动后，ROS2 中已经出现点云相关话题：

```text
/lidar_points
/lidar_imu
/lidar_packets_loss
```

其中 `/lidar_points` 为 XT-16 的主要点云话题。至此，XT-16 实时点云数据已经成功接入 ROS2 系统。
