## 4.7 修复 `/scan` 与 TF 时间戳不一致问题

启动 slam_toolbox 后出现报错：

```text
Message Filter dropping message: frame 'hesai_lidar' at time 1589936675.xxx
for reason 'the timestamp on the message is earlier than all the data in the transform cache'
```

该问题表明：

```text
/scan 时间戳：1589936675.xxx
当前系统/TF 时间：17830656xx.xxx
```

两者差距很大，导致 slam_toolbox 无法在 TF 缓存中查到对应时间的变换。

解决方法：

创建 `/scan` 时间戳修正节点：

```text
~/GO2_Project/go2_bridge_ws/tools/scan_stamp_relay.py
```

功能：

```text
订阅 /scan
发布 /scan_slam
内容基本不变
仅将 header.stamp 改成当前 ROS 时间
保持 frame_id = hesai_lidar
```

运行命令：

```bash
source ~/GO2_Project/go2_ros_env.sh

python3 ~/GO2_Project/go2_bridge_ws/tools/scan_stamp_relay.py \
  --ros-args \
  -p input_topic:=/scan \
  -p output_topic:=/scan_slam \
  -p frame_id:=hesai_lidar
```

检查结果：

```text
/scan_slam Publisher count: 1
frame_id: hesai_lidar
stamp.sec: 1783065853
```

结论：

```text
/scan_slam 时间戳已修正到当前系统时间，可供 slam_toolbox 使用。
```
